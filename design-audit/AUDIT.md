# Grand Trading Webサイト デザイン・UX監査

> 2026-06-25 実装済み: P1項目と主要なP2項目を反映しました。再検証結果と比較画像は `design-audit/after/` にあります。

監査日: 2026-06-25  
対象: 日本語版の全主要ページ、英語版トップ、PC 1280×720、モバイル 390×844  
方式: 実ブラウザ表示、主要導線・中段コンテンツ・モバイルメニュー・アンカー遷移の確認

## 結論

写真、濃紺、ゴールド、明朝体を使ったブランドの方向性はまとまっており、PCのカード・ギャラリー・会社概要表はすでに十分きれいです。一方、公開前に優先して直したいのは次の5点です。

1. モバイルのトップ見出しが単語の途中で改行される。
2. トップから「対象のお客様」へ移動すると、固定ヘッダーが遷移先見出しを隠す。
3. モバイルのアクセス会場カードが2列・11.2px相当になり、住所と地図が小さすぎる。
4. ハンバーガーメニューが `div` のクリック専用で、キーボード操作や状態通知に対応していない。
5. ゴールド文字とパンくずのコントラストが通常文字の基準に届かない。

## 強み

- 写真のトーン、濃紺、ゴールドの組み合わせが全ページで一貫している。
- オークション品目、輸出事例、取引フローは、視線の流れが分かりやすい。
- 会社概要表とお問い合わせフォームは、PCでは読みやすい幅に収まっている。
- 主要画像の欠損、ページ全体の横スクロール、ブラウザのコンソールエラーは確認されなかった。
- 日本語版・英語版とも、PC表示では主要ナビゲーションが破綻していない。

## 優先度つき修正案

### P1: モバイルのヒーロー見出しを正常に折り返す

根拠: `09-home-mobile.png`、`31-en-home-mobile.png` では `CONNECTING` が `CONNECTIN / G` に分割されています。CSSの `word-break: keep-all` と `overflow-wrap: anywhere`、2.8remの文字サイズ、0.1emの字間が同時に効いています。

修正案:

- 768px以下では `font-size: clamp(2rem, 9vw, 2.45rem)`、`letter-spacing: .04em`、`overflow-wrap: normal` にする。
- 見出し内の意図した2行を `<span>` 単位で分け、各spanを `display:block` にする。
- `text-wrap: balance` を併用し、単語途中ではなく語句単位で改行する。

### P1: 固定ヘッダーでアンカー先が隠れる問題を直す

根拠: `29-clients-anchor-mobile.png`。`clients.html#admin` を直接開くと、対象セクション上端は表示領域の -40px、固定ヘッダーは約89pxで、見出しが完全に隠れました。

修正案:

```css
.client-section,
[id] {
  scroll-margin-top: 110px;
}
```

汎用の `[id]` が広すぎる場合は、`#admin, #corp, #trader, #individual` のみに指定します。

### P1: モバイルのアクセスカードを1列にする

根拠: `28-access-cards-mobile.png`。現在は2列、見出し0.8rem、住所0.7remで、実測最小文字サイズは11.2pxでした。住所と地図情報には密度が高すぎます。

修正案:

- 600px以下は `.grid-2 { grid-template-columns: 1fr !important; }`。
- 会場名を16px、住所を14px以上、地図を180〜220px高にする。
- 2列を残すなら住所を省略せず、カード幅が最低280pxになるブレークポイントに限定する。

### P1: モバイルメニューをキーボード対応にする

根拠: `26-mobile-menu-open.png`。見た目と配置は良好ですが、トグルはラベルなしの `div` でDOMのアクセシビリティツリーに現れません。

修正案:

- `<button type="button" aria-label="メニューを開く" aria-controls="global-nav" aria-expanded="false">` に変更。
- 開閉時に `aria-expanded` とラベルを更新し、Escで閉じる。
- 開いたら最初のリンクへフォーカス、閉じたらボタンへ戻す。
- PCのドロップダウンに `.dropdown:focus-within .dropdown-menu` を追加する。

### P1: 文字色のコントラストを上げる

CSS値から算出すると、ゴールド `#B59049` / 白は 2.98:1、パンくず `#888` / `#F0F0F0` は 3.11:1です。通常文字の目安4.5:1を下回ります。

修正案:

- 白背景上の文字専用に `--secondary-text: #80652C`（白に対して約5.5:1）を追加。
- 装飾線や大きな面は現在のゴールドを維持し、本文・小見出し・リンクだけ濃い色へ分ける。
- パンくずは `#595959` 前後に変更する。

### P2: サブページ冒頭の空白を少し縮める

根拠: `02`〜`08`。PCではパンくずのあと、本文見出しまで約120px以上の白地が続き、最初の情報が720px高の画面下端に寄っています。

修正案:

- `.subpage-content` の上下余白をPCで `5rem`、モバイルで `3.5rem` 程度にする。
- トップページの8rem余白はブランド表現として残し、情報ページだけ密度を上げる。

### P2: フェード表示を安全にする

根拠: 読み込み直後は `.fade-up { opacity:0 }` のため、1秒ほど本文が消えます。監査中も最初のキャプチャでは本文が透明な状態でした。JS失敗時は内容が出ない構造です。

修正案:

- JSが有効な時だけ非表示化するクラス設計にする。
- `prefers-reduced-motion: reduce` では変形と遅延を解除し、常に表示する。
- 1秒のトランジションを0.4〜0.6秒へ短縮する。

### P2: ロゴの視認性を安定させる

根拠: `01`〜`08`。青系ロゴが暗い写真に重なると、社名部分のコントラストが不安定です。

修正案:

- ヘッダー上だけ白版ロゴを用意する、またはロゴ背面に薄い白の半透明プレートを敷く。
- スクロール後は現在のカラー版を維持する。

### P2: フォームの状態設計を追加する

根拠: `20-contact-form-desktop.png`、`22-contact-form-mobile.png`。入力欄は整っていますが、必須エラー、送信中、成功、送信失敗の表示がなく、`action="#"` のため実送信は未実装です。

修正案:

- エラーを入力欄直下へ表示し、色だけでなく文言とアイコンでも伝える。
- `:focus-visible` に2px以上の明確なアウトラインを付ける。
- 送信中・完了・失敗の3状態と、プライバシーポリシーへの同意導線を用意する。

### P3: 細部の文章と代替テキストを整える

- トップの `made in japan` は `Made in Japan` など表記ルールを統一。
- `どこでも出品ok‼` は企業サイトのトーンに合わせて `どこからでも出品可能。全国6会場で運営中。` などへ整理。
- トップの「管財物件の提価」は誤記の可能性があるため確認。
- フォーム例の `info@gratrajapan.com` が正式なドメインか確認。
- 輸出事例9枚がすべて同じ `alt="輸出事例"`。内容を説明するaltへ変更するか、装飾画像なら空altにする。
- Google Maps iframeに会場名を含む `title` を追加する。

## 確認できなかったこと

- スクリーンリーダー実機での読み上げ順、各OSの高コントラストモード。
- 実送信先が未設定のため、お問い合わせの完了・失敗フロー。
- 320px幅、200%ズーム、実機Safariでの最終リフロー。
- Google Maps内部のキーボード操作と読み上げ。

スクリーンショットから完全なWCAG適合は断定していません。コード上で確認できた構造と、今回の実ブラウザ表示に基づくリスク評価です。
